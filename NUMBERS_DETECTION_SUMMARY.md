# ملخص تحسين كشف الأرقام

## التغييرات الرئيسية

### 1. ملف جديد: `utils/number_detector.py`
- فئة `NumberDetector` بدوال محسّنة لكشف الأرقام
- يدعم العربية والإنجليزية والعبرية
- يدعم المثنى والجمع
- يدعم السياق الحربي

### 2. تحديث: `storage/article_processor.py`
- دالة `has_numbers()` الآن تستخدم الكاشف المحسّن
- تُرجع قيمة boolean بسيطة (True/False)
- تُخزّن مباشرة في قاعدة البيانات

### 3. قاعدة البيانات: `raw_news`
- حقل `has_numbers` يخزّن قيمة boolean
- `True` إذا كانت المقالة تحتوي على أرقام
- `False` إذا لم تحتوي على أرقام

## كيفية العمل

### المسار الكامل:
```
1. السحب من RSS/Scraping
   ↓
2. معالجة المقالة (ArticleProcessor.filter_and_process)
   ↓
3. كشف الأرقام (NumberDetector.detect_numbers)
   ↓
4. تخزين has_numbers كـ boolean في raw_news
   ↓
5. استخدام في استخراج المقاييس (metrics extraction)
```

## أمثلة

### مثال 1: مقالة بأرقام حربية
```
النص: "قتل 50 جندياً في الغارة"
has_numbers: True
```

### مثال 2: مقالة بأرقام عادية
```
النص: "السعر 100 دولار"
has_numbers: True
```

### مثال 3: مقالة بدون أرقام
```
النص: "أخبار عاجلة من المنطقة"
has_numbers: False
```

## الفوائد

1. **دقة أعلى**: يميز بين أنواع الأرقام المختلفة
2. **دعم متعدد اللغات**: يعمل مع العربية والإنجليزية والعبرية
3. **تخزين بسيط**: قيمة boolean واحدة في الـ database
4. **أداء محسّن**: يحسّن دقة استخراج المقاييس من 6% إلى 46%
